# CRAFTAHUB Mini Packaging Generator — пошаговая инструкция для человека без опыта программирования

## Что лежит в этой папке
Это **рабочий starter MVP** инструмента:
- 15 шаблонов есть в интерфейсе
- масштаб 1:6 и 1:12 есть
- загрузка PNG/JPG/SVG/PDF есть
- заливка частей цветом есть
- move / resize / rotate есть
- duplicate to another panel есть
- print export SVG / PDF / PNG / JPG есть
- Cricut SVG export есть

Важно:
- это **первая рабочая версия**, а не окончательно отполированный production-финал;
- сложные шаблоны вроде round / elliptical / polygonal / cone уже заведены и работают как MVP, но перед публичным запуском их обязательно надо протестировать на реальной печати;
- PDF artwork для preview растеризуется через pdf.js.

# ЧАСТЬ 1. Самый простой путь: выложить проект без программирования

## Шаг 1. Создай аккаунт на GitHub
1. Открой сайт GitHub.
2. Нажми **Sign up**.
3. Создай аккаунт.
4. Подтверди email.

## Шаг 2. Создай новый repository
1. После входа на GitHub нажми в правом верхнем углу **+**.
2. Выбери **New repository**.
3. В поле **Repository name** вставь:

```text
craftahub-mini-packaging-generator
```

4. Выбери **Public** или **Private**.
   - Для старта лучше **Private**.
5. Нажми **Create repository**.

## Шаг 3. Загрузи файлы проекта на GitHub
1. Открой только что созданный repository.
2. Нажми **uploading an existing file**.
3. Открой на своем компьютере папку:

```text
craftahub-mini-packaging-generator
```

4. Перетащи **все файлы и папки** из нее в окно GitHub.
5. Внизу страницы в поле commit message напиши:

```text
Initial upload of Craftahub generator
```

6. Нажми **Commit changes**.

Готово: код уже в GitHub.

# ЧАСТЬ 2. Размещение на Vercel

## Шаг 4. Создай аккаунт на Vercel
1. Открой Vercel.
2. Нажми **Sign Up**.
3. Выбери вход через **GitHub**.
4. Разреши доступ к GitHub.

Vercel умеет деплоить проекты из Git, а у Vite есть официальный путь деплоя на Vercel. Также Vercel позволяет конфигурировать проект через `vercel.json`, где можно задать заголовки для встраивания и безопасности. citeturn533748search1turn533748search8turn533748search10

## Шаг 5. Импортируй проект в Vercel
1. Внутри Vercel нажми **Add New...**
2. Выбери **Project**.
3. Найди репозиторий:

```text
craftahub-mini-packaging-generator
```

4. Нажми **Import**.
5. В блоке настроек ничего сложного менять не нужно.
6. Нажми **Deploy**.

Подожди завершения сборки.

После этого Vercel даст тебе ссылку вида:

```text
https://craftahub-mini-packaging-generator.vercel.app
```

Сохрани эту ссылку.

# ЧАСТЬ 3. Встраивание в Wix

Wix поддерживает встраивание внешнего сайта или HTML/iFrame-элемента через Embed. В документации Wix это делается через **Embed Code** или **Embed a site**. citeturn533748search0turn533748search3turn533748search14

## Шаг 6. Открой редактор Wix
1. Зайди в свой аккаунт Wix.
2. Открой сайт **craftahub.com**.
3. Нажми **Edit Site**.

## Шаг 7. Создай подстраницу в How-to
1. Слева открой меню **Pages & Menu**.
2. Найди страницу **How-to**.
3. Создай новую подстраницу внутри How-to.
4. Назови ее, например:

```text
Mini Packaging Generator
```

## Шаг 8. Добавь iframe / embed
Есть 2 варианта:

### Вариант A — проще всего
1. В редакторе нажми **Add Elements**.
2. Найди раздел **Embed Code**.
3. Выбери **Embed a site**.
4. Вставь ссылку Vercel, например:

```text
https://craftahub-mini-packaging-generator.vercel.app
```

5. Растяни блок почти на всю ширину страницы.
6. Сделай большую высоту, например 1500–1800 px.

### Вариант B — через HTML iframe
1. В редакторе нажми **Add Elements**.
2. Выбери **Embed Code**.
3. Выбери **Embed HTML**.
4. Вставь этот код:

```html
<iframe
  src="https://craftahub-mini-packaging-generator.vercel.app"
  style="width:100%;height:1600px;border:0;overflow:hidden;"
  loading="lazy"
  referrerpolicy="strict-origin-when-cross-origin"
  allow="clipboard-write"
></iframe>
```

5. Нажми **Apply**.
6. Растяни блок по ширине страницы.

## Шаг 9. Проверь превью
1. Нажми **Preview**.
2. Открой страницу.
3. Проверь:
   - открывается ли приложение,
   - работает ли выбор шаблона,
   - меняются ли поля,
   - грузится ли artwork,
   - скачиваются ли файлы.

## Шаг 10. Опубликуй сайт
1. Нажми **Publish**.
2. Открой публичный сайт.
3. Еще раз проверь подстраницу.

# ЧАСТЬ 4. Что делать, если Wix не показывает приложение

Если внутри Wix приложение не отображается, проверь:

## 1. Правильный ли URL
Открой ссылку Vercel отдельно в новой вкладке.
Если там приложение открывается — ссылка верная.

## 2. Заголовки на Vercel
В проекте уже есть файл:

```text
vercel.json
```

В нем добавлен `Content-Security-Policy` с `frame-ancestors`, чтобы приложение можно было встроить в Wix iframe, и добавлен CORS-заголовок. На практике именно `frame-ancestors` в CSP является современным механизмом контроля встраивания, а не `X-Frame-Options: ALLOWALL`. citeturn533748search2turn533748search8

Если после первой публикации ты меняла этот файл, нужно:
1. снова загрузить изменения в GitHub;
2. дождаться нового деплоя в Vercel.

# ЧАСТЬ 5. Как обновлять инструмент дальше

Каждый раз логика одна и та же:

1. меняешь файлы проекта;
2. загружаешь новые версии в GitHub;
3. Vercel автоматически пересобирает проект;
4. Wix iframe подтягивает уже новую версию.

То есть в Wix код каждый раз почти не надо трогать.

# ЧАСТЬ 6. Самая простая схема редактирования без терминала

Если ты не хочешь ничего устанавливать на компьютер:

1. Открываешь GitHub repository.
2. Нажимаешь нужный файл.
3. Нажимаешь значок карандаша **Edit**.
4. Вносишь изменения прямо в браузере.
5. Нажимаешь **Commit changes**.
6. Ждешь 1–2 минуты, пока Vercel обновит сайт.

Это самый простой режим для человека без опыта.

# ЧАСТЬ 7. Что именно редактировать чаще всего

## 1. Если нужно поменять тексты интерфейса
Открывай файл:

```text

```

## 2. Если нужно поменять цвета, отступы, внешний вид
Открывай файл:

```text
src/styles.css
```

## 3. Если нужно поменять шаблоны, параметры, геометрию
Открывай файл:

```text
src/templates.ts
```

Вот это самый важный файл проекта.

# ЧАСТЬ 8. Как запустить локально на компьютере (не обязательно)

Этот блок нужен, только если захочешь тестировать до загрузки на GitHub.

## Шаг 1. Установи Node.js
1. Открой официальный сайт Node.js.
2. Скачай **LTS version**.
3. Установи обычным способом.

## Шаг 2. Установи Visual Studio Code
1. Открой сайт VS Code.
2. Скачай версию для Mac.
3. Установи.

## Шаг 3. Открой папку проекта в VS Code
1. Запусти VS Code.
2. Нажми **File → Open Folder**.
3. Выбери папку:

```text
craftahub-mini-packaging-generator
```

## Шаг 4. Открой Terminal внутри VS Code
1. В верхнем меню нажми **Terminal**.
2. Нажми **New Terminal**.

Внизу откроется окно терминала.

## Шаг 5. Введи команды по очереди
Скопируй и вставь:

```bash
npm install
```

Дождись окончания.

Потом:

```bash
npm run dev
```

После этого VS Code покажет локальную ссылку, обычно что-то вроде:

```text
http://localhost:5173
```

Открой ее в браузере.
